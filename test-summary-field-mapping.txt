# CSV Field Mapping Route Tests Summary

## File Created
- tests/unit/csv-field-mapping-route.test.ts

## Test Coverage Overview

Total Test Cases: 35
Total Describe Blocks: 20

### Test Organization

#### 1. POST - SUGGEST MODE (without confirmedMapping)
- **Group 1: Authentication & Validation** (2 tests)
  - Missing uploadId returns 400
  - Non-existent upload record returns 404

- **Group 2: Status Updates** (1 test)
  - Upload status updated to "mapping" on suggestion request

- **Group 3: Mapping Suggestions** (4 tests)
  - Returns suggested mapping on success
  - Includes all CSV headers as keys
  - Fallback to pattern matching when no LLM keys
  - Handles CSV file read errors gracefully

#### 2. POST - IMPORT MODE (with confirmedMapping)
- **Group 1: Mapping Validation** (2 tests)
  - Returns 400 when validation fails (missing item)
  - Accepts valid mapping with required fields

- **Group 2: Data Import & Parsing** (2 tests)
  - Reads CSV file from disk
  - Parses all rows from CSV file

- **Group 3: Row Validation** (3 tests)
  - Rejects rows missing required "item" field
  - Rejects rows with invalid/missing date
  - Rejects rows with invalid/missing quantity

- **Group 4: Database Insertion** (4 tests)
  - Inserts valid rows into transactions table
  - Inserts with correct locationId from upload record
  - Sets source to "csv" for all imported rows
  - Proper transaction value creation

- **Group 5: Partial Failure Handling** (2 tests)
  - Imports valid rows and collects errors for invalid rows
  - Returns errors array with row numbers and messages

- **Group 6: Status Management** (3 tests)
  - Sets upload status to "complete" on successful import (no errors)
  - Sets upload status to "complete" on partial success (some errors)
  - Sets upload status to "error" when ALL rows fail

- **Group 7: File Cleanup** (2 tests)
  - Deletes temp CSV file after successful import
  - Deletes temp CSV file even after import failures

- **Group 8: Response Structure** (2 tests)
  - Returns response with status, rowsImported, and mapping
  - Stores confirmed mapping as JSON in upload record

#### 3. GET - Retrieve Existing Mapping
- **Group 1: Query Parameter Validation** (1 test)
  - Returns 400 when uploadId query param missing

- **Group 2: Upload Record Lookup** (2 tests)
  - Returns 404 when upload not found
  - Returns upload record when found

- **Group 3: Existing Mapping Retrieval** (2 tests)
  - Returns existing mapping if already confirmed
  - Returns success without mapping if not yet confirmed

- **Group 4: Error Handling** (1 test)
  - Returns 500 on database error

#### 4. Error Scenarios & Edge Cases
- **Group 1: Error Handling & Edge Cases** (3 tests)
  - Handles JSON parsing error in POST body
  - Handles missing CSV file when reading for suggestions
  - Handles unexpected database insertion errors

## Mocking Strategy
- Database operations mocked with Drizzle-like chains
- CSV parser mocked to return controlled data
- Field mapper functions mocked for suggestion/validation/mapping
- All database inserts/updates/selects intercepted via vi.mock()

## Test Validation Scenarios
The tests include:
- Valid single row imports
- Multiple valid rows
- Partial failures (mix of valid and invalid rows)
- Total failure (all rows invalid)
- Edge cases (missing files, DB errors)
- Proper error reporting with row numbers
- Status transitions throughout import lifecycle
